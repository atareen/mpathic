Overview¶

evaluate_model is a program within the sortseq_tools package which uses an energy model to calculate the predicted energies of each sequence in a library. It then adds a ‘val’ column to the tabular data file you input containing the predicted energies.

After you install `sortseq_tools`_, this program will be available to run at the command line.

Command-line usage¶

usage: sortseq_tools evaluate_model [-h] [-m MODEL] [-i I] [-o OUT] [-l LEFT]
                                    [-r RIGHT] [-f]
Options:
-m, --model Name of file containing model dataframe. Required.
-i=False, --i=False
 Name of file containing dataset dataframe. Defaults to stdin.
-o, --out Name of file to write output dataframe. Defaults to stdout.
-l, --left Seq position at which to align the left-side of the model. Defaults to position determined by model dataframe.
-r, --right Seq position at which to align the right-side of the model. Defaults to position determined by model dataframe.
-f=False, --fast=False
 Output is a little harder to read, but is written much faster.

Examples¶

The input table to this function should be a standard data file for the sortseq package. The furthest right column should contain sequence identity. If the sequences DNA the column should be labeled ‘seq’, if they are RNA, the column should be labeled ‘seq_rna’, and if they are amino acid sequences they should be labeled ‘seq_pro’. Other columns are optional, but if you have counts columns they must follow the standard layout (‘ct’ on the far left, then ‘ct_K’ for each bin). Order of columns does matter.

Example Input Data Table:

ct    seq
13    ACTAG
5     AGGTA
...

Example Input Model:

pos    val_A    val_C    val_G     val_T
0      .04      -.3      -.2       .15
1      .2       .1       -.44      .05
...

Example Output Table:

ct     val    seq
10     -1.4   ACGAT
5      -.5    GGACC
...
